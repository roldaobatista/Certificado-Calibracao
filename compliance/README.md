# compliance/ — Governança e evidência

Camada regulatória de 1ª classe (P1-3). Estrutura canônica:

- `normative-packages/` — pacotes normativos versionados e assinados (P0-2).
- `validation-dossier/` — `requirements.yaml`, `traceability-matrix.yaml`, evidência por execução (P0-3).
- `release-norm/` — veredito por release assinado por `product-governance` (P0-6).
- `legal-opinions/` — pareceres jurídicos datados.
- `audits/metrology|legal|code/` — pareceres dos 3 auditores externos (P0-12).
- `approved-claims.md` — claims aprovados para copy (P0-5).
- `guardrails.md` — 7 gates de `harness/05-guardrails.md` (P0-4).
- `runbooks/` — runbooks de recuperação regulatória e calendário de drills (P0-9).
- `verification-log/` — registros de propagação L0-L5 e re-auditorias (P0-10).
- `cloud-agents-policy.md` — política de Tier 3 (P1-2).
- `budget-log/` — consumo de tokens/custo por sessão (P0-7).
- `sessions-log/` — handoff de sessões para `/resume` e continuidade entre agentes (P0-13).
- `roadmap/` — roadmap V1-V5 com fatias verticais, gates de saída e trilhas transversais excluídas do roadmap vertical (P1-4).
